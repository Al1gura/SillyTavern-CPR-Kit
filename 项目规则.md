## 酒馆机制关键约束

| 约束 | 影响 |
|:---|:---|
| **世界书靠关键词触发** | uid 的 `key` 数组决定什么时候插入。key 不能太泛（消耗token预算），不能太偏（触发不了） |
| **token 预算有限** | 世界书条目要精简。`content` 过长会挤掉对话历史 |
| **深度/优先级控制** | `order` 越大越靠后（影响力越大），`depth` 控制扫描多少条消息 |
| **骰子走 ACU 插件** | 不是酒馆内置。`骰子规则/` 里的 JSON 是 ACU 高级骰子预设，`CPR检定_v2.3` 负责技能判定，`CPR伤害_v1.2` 负责伤害 |
| **提示词分层注入** | 系统提示 → 世界书条目 → 角色定义 → 对话历史 → 用户消息。你的 `提示词/` txt 文件目前是通过 Izumi 预设的扩展提示注入 |
| **字段名敏感** | 世界书 json 里的字段名（如 `uid`, `key`, `order`, `depth`）不能乱改，酒馆按这些字段解析 |
